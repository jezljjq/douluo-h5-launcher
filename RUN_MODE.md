# 运行模式说明

**当前版本：前台串行模式**

---

## 1. 支持的运行模式

### 单账号运行

- 触发：GUI 下拉框选择具体账号 → 点击"单账号运行"
- 行为：运行选中账号的完整流程（OCR → 登录 → 校验）
- 适用：调试单个账号、验证流程正确性

### 当前层串行

- 触发：GUI 选择层级（第一层/第二层/第三层/第四层）→ 点击"当前层串行"
- 行为：逐个串行运行该层全部 8 个账号
- 顺序：收藏 1 → 2 → 3 → ... → 8
- 适用：批量登录同一层账号

### 全部串行

- 触发：点击"全部串行"
- 行为：逐个串行运行所有已加载账号（最多 32 个）
- 顺序：第一层 1-8 → 第二层 1-8 → 第三层 1-8 → 第四层 1-8
- 适用：全量批量登录

---

## 2. 运行特性

| 特性 | 说明 |
|------|------|
| 运行模式 | 源码模式子进程隔离 / exe 模式同进程直接调用 |
| 实时状态 | STATUS/PASSPORT 实时推送 → GUI 表格 + 颜色标签 |
| 失败重试 | 任意步骤失败自动重试 1 次（始终走完整浏览器流程：关旧→开新→公告→按钮→输入→确认→校验） |
| 停止响应 | 200ms 内响应（所有等待可中断，不再等步骤完成） |
| 已登录跳过 | 500ms 二次确认，快速跳过已登录窗口 |
| 失败保留 | 失败截图保留到 `debug_ocr/_error/` + 浏览器窗口保留 |
| 文件日志 | `logs/run_时间.log` 完整日志 + GUI 简洁日志 |
| 残留清理 | 每个账号完成后 taskkill chromium 残留进程 |

---

## 3. 运行限制

| 限制 | 说明 |
|------|------|
| **前台模式** | 运行期间会短暂移动鼠标，请勿操作 |
| **严格串行** | 不支持并发，账号逐个执行 |
| **非后台** | 窗口必须在屏幕上可见 |
| **不可遮挡** | 浏览器窗口被遮挡时点击可能失败 |

---

## 4. 停止任务

- 点击"停止任务"按钮
- 当前正在执行的账号会尽可能完成当前步骤后停止
- 后续排队的账号不再执行

---

## 5. 运行方式

### 从源码运行

```powershell
cd D:\Ai\codex\上号器
python main.py
```

### 从打包 exe 运行

```powershell
.\scripts\build_exe.bat          # 先打包
cd dist\斗罗大陆H5上号器
.\斗罗大陆H5上号器.exe            # 运行
```

**注意**：exe 模式下单账号和批量均走同进程直接调用（`sys.frozen` 检测）。Dm 点击仍需 `py -3.14-32`。详见 [BUILD.md](BUILD.md)。
