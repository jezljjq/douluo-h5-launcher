# GUI 状态流转说明

**GUI 框架：Tkinter + Treeview**
**状态同步：子进程 stdout → Popen 逐行读取 → UI 队列 → 主线程更新**

---

## 1. 状态值完整列表

| 状态 | 含义 | 标签颜色 | 触发时机 |
|------|------|----------|----------|
| `未开始` | 账号等待执行 | 默认 | 读取收藏夹后初始化 / 串行队列中 |
| `OCR中` | 正在从登录程序窗口提取通行证 | 蓝色 | 步骤1 开始 |
| `已提取通行证` | OCR 成功获取 8 位 hex | 蓝色 | 步骤1 完成 |
| `打开中` | 正在启动浏览器打开游戏页 | 蓝色 | 步骤2 开始 |
| `关闭公告` | 正在关闭游戏内公告弹窗 | 蓝色 | 步骤3 开始 |
| `已关闭公告` | 公告关闭完成 | 蓝色 | 步骤3 完成 |
| `输入中` | 正在向弹窗输入通行证 | 蓝色 | 步骤5 开始 |
| `已输入通行证` | 通行证输入完成 | 蓝色 | 步骤5 完成 |
| `成功` | 登录校验通过 | 绿色 | 步骤7 QR 码消失 |
| `失败` | 登录失败 / OCR 失败 / 异常 | 红色 | 任何异常且重试已用尽 |
| `重试` | 失败后自动重试 | 橙色 | 第一次失败后自动触发 |

## 3. 表格列更新机制

### "状态"列

- 子进程通过 `print("STATUS:<状态文本>", flush=True)` 输出
- 主进程 Popen 逐行读取，匹配 `STATUS:` 前缀
- 通过 `_queue_status()` 加入 UI 队列
- 主线程 `_drain_ui_queue()` 调用 `_set_status()` 更新 Treeview
- 同步更新颜色标签（blue/green/red/orange）

### "本次通行证"列

- 子进程通过 `print("PASSPORT:<hex>", flush=True)` 输出
- 主进程匹配 `PASSPORT:` 前缀
- 通过 `_queue_passport()` → `_set_passport()` 更新

## 4. 线程模型

```
主线程（Tkinter）
  ├── UI 事件循环
  ├── _drain_ui_queue() 每 100ms 轮询
  │    处理: log / status / passport / passport_prompt
  └── 按钮点击 → 启动 worker 线程

worker 线程（daemon）
  └── _serial_worker()
        ├── sys.frozen 检测运行模式：
        │     exe 模式 → AccountRunner 同进程直接调用
        │     源码模式 → Popen 子进程（每个账号独立）
        ├── _queue_log / _queue_status / _queue_passport
        └── 线程安全：所有 UI 更新通过 queue 回到主线程
```

---

## 5. 关键约束

- Tkinter 不是线程安全的，所有 widget 操作必须在主线程
- worker 线程仅通过 `ui_queue.put()` 发送更新
- 源码模式 Popen 阻塞读取 stdout 直到子进程退出
- exe 模式 AccountRunner 在 worker 线程内同步执行
- 串行运行时完整日志写入 `logs/run_时间.log`
