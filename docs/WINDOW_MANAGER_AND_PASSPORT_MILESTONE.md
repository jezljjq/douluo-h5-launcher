## 4. 通行证复制优先方案说明

当前通行证获取优先级：

1. 第一优先级：双击选中 8 位通行证，`Ctrl+C` 复制，读取剪贴板。
2. 第二优先级：复制失败时才进入 OCR 兜底。

OCR 不再作为第一优先级。原因是 OCR 存在单字符误识别风险，例如：

- 真实：`4425cbaa`
- 错误：`4425ebaa`
- 错误位置：`c` 被识别为 `e`

复制成功时直接使用复制结果。复制失败时才进入 OCR。OCR 低置信度时不得继续输入通行证。OCR 存在 `c/e` 等混淆时必须失败或进入人工处理，不能假成功。

## 5. OCR 兜底规则说明

- 已删除或禁用“低票数接受最佳候选”逻辑。
- 新增失败类型：
  - `OCR_LOW_CONFIDENCE`
  - `OCR_AMBIGUOUS_CHAR`
- OCR 兜底必须输出：
  - 候选结果
  - 候选票数
  - 字符级投票
  - 存疑位置
  - 是否接受
  - 失败类型
- 原则：宁可真实失败，也不要错误输入通行证。

### 快速登录缓存与统一校验

- 缓存文件：`debug_ocr/passport_dialog_pos_cache.json`
- 缓存 key：浏览器真实 viewport，例如 `960x720`
- 缓存内容：`button`、`input`、`confirm`、`updated_at`
- 合并链路：`click 通行证按钮 → wait → click 输入框 → type 通行证 → click 确认`
- 当前层串行 / 全部串行使用快速提交 + 统一校验。
- 单账号运行仍保留完整校验。
- `重新次数` 用于限制统一校验和失败重登次数。
- 统一校验提前全部成功时，不继续执行剩余校验轮次。

安全规则：

- `qr_page` 不能判成功。
- `unknown` 不能判成功。
- 截图失败不能判成功。
- 只有明确 `logged_in` 才成功。
- 失败账号只重登失败账号，不全量重跑。

### 停止与关闭清理机制

停止任务机制已修复：

- 点击“停止任务”后，不再只是设置 `stop_event`。
- 会强制终止当前账号运行子进程。
- 会清理 `dm_click_helper.py` 子进程。
- 会清理本次 Playwright/Chromium 相关进程。
- 不再继续后续账号。

关闭程序机制已修复：

- 点击窗口右上角关闭时，会先执行停止任务和子进程清理。
- GUI 关闭后，不允许残留子进程继续移动鼠标。
- 避免 `_drain_ui_queue` 在窗口关闭后继续 `after` 回调。

安全规则：

- 运行中点击停止，必须立即停止。
- 关闭程序前必须清理子进程。
- 禁止只做协作式 `stop_event`，不清理实际子进程。
- 禁止误杀所有 `python.exe`，只清理本项目相关子进程。

- 窗口管理参数记忆可能存在未覆盖的边界值或损坏配置文件场景，需要现场验证。
