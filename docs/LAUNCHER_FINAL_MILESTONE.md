# 上号器快速登录缓存与统一校验收尾总结

更新时间：2026-05-17

## 1. 当前新增能力

- 通行证弹窗坐标文件缓存已接入。
- 缓存文件：`debug_ocr/passport_dialog_pos_cache.json`
- 缓存 key 使用浏览器真实 viewport，例如 `960x720`。
- 缓存内容包括：
  - 通行证按钮坐标
  - 输入框坐标
  - 确认按钮坐标
  - 更新时间
- 缓存用于跨账号、跨源码模式子进程复用弹窗坐标。
- 命中缓存后使用合并 Dm chain。

合并链路：

```text
click 通行证按钮
→ wait
→ click 输入框
→ type 通行证
→ click 确认
```

## 2. 批量流程

- 单账号运行仍保留原完整校验逻辑。
- 当前层串行 / 全部串行使用快速提交 + 统一校验。
- 第一轮逐个完成：获取通行证、打开页面、关闭公告、点击通行证、输入通行证、点击确认。
- 第一轮提交完成后状态为 `待复核`，不能直接标记成功。
- 统一校验逐个检查登录程序窗口状态。
- 失败账号只重登失败账号。
- `重新次数` 用于限制统一校验和失败重登轮数。
- 如果提前全部成功，不继续执行剩余校验轮次。

## 3. 安全规则

- `qr_page` 不能判成功。
- `unknown` 不能判成功。
- 截图失败不能判成功。
- 只有明确 `logged_in` 才能成功。
- 不允许恢复假成功逻辑。
- 不允许放宽 OCR 兜底规则。
- 不允许回退通行证复制优先策略。

如需回滚快速批量策略：

- 保留单账号 `run_game_flow()` 完整校验逻辑。
- 当前层串行 / 全部串行可退回逐账号完整校验。
- 删除或忽略 `debug_ocr/passport_dialog_pos_cache.json` 不影响安全路径。
- 回滚时不得放宽 `qr_page`、`unknown`、截图失败的失败判定。
